# RAG 技术简介

## 什么是 RAG

RAG（Retrieval-Augmented Generation）是一种结合检索和生成的 AI 技术架构。它通过从外部知识库中检索相关信息，然后将其注入到大语言模型的上下文中，从而生成更准确、更有依据的回答。

## RAG 的核心流程

1. **文档处理**：将原始文档进行切片，转换为向量表示
2. **向量存储**：将向量存入向量数据库（如 Chroma、Milvus）
3. **检索阶段**：用户提问时，将问题转换为向量，检索最相似的文档片段
4. **生成阶段**：将检索到的文档片段和用户问题一起发送给 LLM，生成最终回答

## RAG 的优势

- **知识更新**：无需重新训练模型，只需更新知识库
- **可追溯性**：回答可以引用具体来源
- **成本效益**：比微调大模型更经济
- **隐私保护**：敏感数据可以本地存储

## 常见的向量数据库

| 数据库 | 特点 |
|--------|------|
| Chroma | 轻量级，适合开发测试 |
| Milvus | 分布式，适合生产环境 |
| Pinecone | 云托管，开箱即用 |
| Weaviate | 支持混合检索 |

## 文本切分策略

- **固定长度切分**：按字符数切分，简单但可能破坏语义
- **递归切分**：按段落、句子、词逐级切分
- **语义切分**：根据语义相似度切分

## 本项目的技术栈

- **后端**：FastAPI + LangChain + Chroma
- **前端**：Vue 3 + Element Plus
- **数据库**：PostgreSQL + Redis
- **模型**：mimo（对话）+ qwen3（Embedding）
